## 正文

最近做了一个公考备考用的小工具，叫 **上岸笔记**。

主要想解决刷题之后的整理问题。

平时做完题，经常还要抄题干、标原因、整理错题本、过几天再翻出来二刷。这个过程比较碎，也很容易拖着不做。

这个工具的用法大概是：

1. 把错题截图发给 agent
2. agent 识别题目内容和答案信息
3. 自动归到对应模块，比如判断推理 / 资料分析
4. 记录错误原因，比如粗心、知识点不会、时间不够
5. 写进本地错题本
6. 需要的时候导出 Excel

它按通用 agent skill 的方式写的，Trae、Cursor、opencode、Hermes、OpenClaw，或者其他能读说明、跑脚本、调用图片模型的 agent，都可以接。

目前已经验证过这些流程：

- 自然语言打卡：比如“言语4错，判断8错，数量5错，资料3错”
- 截图识别后整理成结构化错题
- 导出 Excel 错题本，包含“错题本”和“每日记录”两个 Sheet
- 晚上自动总结当天弱项
- 按艾宾浩斯遗忘曲线排期，到点提醒你复习（记得就拉长间隔，忘了就重来）

数据默认存在自己电脑本地。如果主动配置飞书，也可以同步到自己的飞书文档里看。

适合：

- 每天刷行测，但经常懒得整理错题的人
- 图形推理、资料分析截图很多的人
- 想把错题导出给 Kimi / ChatGPT 继续分析的人
- 想让复习提醒自动一点的人

截图识别需要当前 agent 配置支持图片输入的多模态模型。没有图片能力的话，也可以用文字方式记录。
