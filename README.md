<h1 align="center">上岸笔记 · 行测备考追踪</h1>

<p align="center">
  把每天的套题截图或一句成绩汇报发给你的 AI agent，<br>
  它帮你归档错题、自动定位知识点、分析弱项、定时提醒复习——<br>
  所有数据存在你本地，不上传任何云端。
</p>

---

## 这是什么

备考行测，每天刷套题是标准动作。但真正提分的部分——**错题归类、原因分析、二刷追踪**——因为太麻烦，往往被跳过。

上岸笔记想解决的就是这件事：**让"做完题之后的整理"几乎不需要努力**。你只需把错题截图发出来，或发一句「今天判断推理错了 8 道」，剩下的它来做。

> 它不是绑定某个 App 的工具，而是一个 **Agent Skill**：任何能读 `SKILL.md`、跑 Node.js 脚本、读写本地文件的 AI 助手都能用（Cursor / Trae / CodeBuddy / Claude Code / OpenClaw 等）。

## 它能帮你做什么

| 功能 | 一句话说明 |
|------|-----------|
| **截图自动归档** | 发 1~N 张错题截图，agent 多模态识别题目内容、科目、错因、知识点，自动建卡 |
| **文字快速录入** | 发「资料-增长率-公式不熟」一行搞定一道题，支持多行批量 |
| **自然语言打卡** | 发「今天判断 8 错、资料 3 错、粗心居多」只记模块级成绩 |
| **自动定位知识点** | 每道错题自动挂到 472 节点的行测知识框架的对应考点下（可人工核对修正） |
| **HTML 知识本** | 生成可交互的知识树网页，错题卡片直接挂在对应知识点旁边，带全文搜索 |
| **Excel 错题本** | 一键导出，含知识路径列、原图嵌入，可丢给 Kimi/ChatGPT 进一步分析 |
| **晚间智能总结** | 每天定时推送当天成绩汇总 + 最弱模块 + 一条具体建议（含错因分析） |
| **遗忘曲线复习提醒** | 按艾宾浩斯遗忘曲线（1→2→4→7→15→30→60→120 天）自动排期，到点推送让你自评，「记得」拉长间隔、「忘了」重置，走完周期自动标记「已掌握」 |

### 内置行测知识框架（472 个考点）

覆盖言语理解、判断推理、资料分析、数量关系四大模块的全部核心考点，每个考点含完整讲解、公式技巧、配图。每道错题提交后**自动定位到对应考点**——这是整个工具区别于普通「错题本 App」的核心。

### 设计原则

- **对话即操作**——所有功能通过聊天完成，不用打开 App、不用手动填表
- **本地优先**——错题数据（含截图）默认在你电脑的 `~/.apexnotes/` 里，不经过任何第三方服务器
- **失败软降级**——截图识别失败会提示你复制文字，功能不中断
- **数据独立**——项目文件夹和你的错题数据**分离**：分享项目时只分享文件夹，对方安装后建立自己的空错题库，不会包含你的错题

## 快速开始（3 步）

### 1. 安装

```bash
cd ApexNotes
bash install.sh
```

脚本会自动完成：检测 Node.js（≥18）→ `npm install`（sharp，图片重压优化）→ 安装 Python 依赖 openpyxl+Pillow（Excel 导出需要）→ 建数据目录 + **初始空错题库** + 解析 472 节点知识框架。

> 手动安装也行：`pip install -r requirements.txt` → `node scripts/init_demo.js`（仅需要 Excel 导出时）

**依赖清单：**

| 依赖 | 类型 | 用途 | 是否必需 |
|------|------|------|----------|
| Node.js ≥18 | 运行时 | 核心脚本（录入 / 复习 / HTML 导出） | 必需 |
| `sharp` (npm) | Node 包 | 错题图批量重压（800px/jpeg80，省空间） | 可选（不装也能跑，图更大） |
| Python 3.x | 运行时 | 仅 Excel 导出时调用 | 仅 xlsx 导出必需 |
| `openpyxl` | Python 包 | 生成 .xlsx 错题本 | 仅 xlsx 导出必需 |
| `Pillow` | Python 包 | openpyxl 嵌入图片依赖（`XLImage`） | 仅 xlsx 导出必需 |

> 数据存储完全独立于项目文件夹（见下文），克隆后 `bash install.sh` 即建立你自己的空错题库，不含他人错题。

### 2. 交给你的 agent

把项目里的 `SKILL.md` 交给你的 AI 助手读取（不同平台方式见 `PLATFORM_GUIDE.md`，比如 Cursor 放 `.cursor/skills/`、CodeBuddy 放 `~/.workbuddy/skills/`）。

### 3. 开始录入错题

直接发一句试试：

```
今天判断推理错了 8 道，资料分析错了 3 道，粗心居多
```

或者更直接——**发一张错题截图**。

> ✅ 初始是**空错题库**，所有错题由你自己的备考逐步积累，不预置任何演示数据。

## 怎么录入错题（核心功能）

你发消息后，agent 会根据内容自动走最合适的一条通道。

### 通道 A · 截图批量（最推荐）

直接发 1~N 张错题截图。agent 用多模态能力逐张读图，自动提取题目内容、科目、题型、错因推测、知识点，然后对每道做精确定位。

> 没有多模态模型？把题目文字复制发过来，用文字方式录入，功能完全相同。

### 通道 B · 快捷格式（单行 / 多行批量）

适合你已经知道错在哪、只想快速记账的情况：

```
判断-逻辑判断-粗心
资料-增长率-公式不熟
言语-主旨概括-时间不够
```

每行一道题，格式为 `模块-题型-错因`。支持一次发好几行，批量录入。

**想让这道题复习时能看到题面？** 在标记后换行粘贴题干即可（第一行标记、后续行当真题干）：

```
资料-增长率-公式不熟
某省2025年GDP增长率为12%，上年为10%，求隔年增长率？
A.10% B.12% C.20% D.22%
```

这样录入的题在 HTML 知识本上会显示题干文字（无图时），复习时能直接重做；不贴题干也行，只是那道题在知识本上是空卡片（只能用于统计，不能重做）。

### 通道 C · 自然语言打卡

只想记个成绩、不想展开具体题：

```
今天判断错了8道，资料错了3道，粗心居多
```

这类只记录**模块级成绩**（错题数、情绪、错因分布），不展开具体错题——适合快速日报。

### 录入时你要确认什么：核对归类

每道题定位后，agent 会展示一张确认卡片，**核心目的是让你核对「归类对不对」**（比如一道图形推理的「平移」题，别被归到「对称」上去）：

```
【第 1 / 3 道】
归类 → 判断推理 > 图形推理 > 位置规律 > 平移（精确匹配）
关键词：平移、位置变化
错因：粗心
────────────────
回复「对」确认 / 「归类改成XX」修正归类 / 「错因改成XX」修正错因 / 「跳过」丢弃
```

- **归类路径**：这道题挂到了知识框架的哪个考点。这是最该核对的——引擎可能归错，你一眼能发现
- **关键词**：引擎用什么关键词去定位的，帮你判断归类是否合理
- **错因**：4 种分类之一——`知识点不会` / `粗心` / `时间不够` / `概念混淆`
- 回复「对」确认入库；回复「归类改成平移」或「错因改成知识点不会」可修正；回复「跳过」丢弃

确认后错题写入本地错题库，并自动挂载到知识本对应考点下。

## 看你的错题本：HTML 知识本

录入错题后，生成可视化知识本网页：

```bash
python scripts/build_unified_html.py
```

生成 `国考行测-上岸笔记.html`（单文件，双击浏览器打开）。布局：

- **左侧**：行测知识树导航（四大模块 → 考点逐层展开）
- **中间**：知识点图文讲解（公式、技巧、注意事项、配图）
- **右侧**：挂在该考点下的**错题卡片**，带「错因：xxx」标签 + 复习状态标签（待二刷/复习中/已掌握），不再显示题目 ID

顶部有**全文搜索框**，输入关键词可跨整本知识点 + 错题定位跳转。

> 每录入一批新错题，重新跑上面这条命令即可刷新知识本。复习状态在 JSON 源文件中维护，HTML 为只读展示。

## 导出 Excel 错题本

```
导出错题本
```

生成包含两个 Sheet 的 `.xlsx`：

## 复习与提醒

### 遗忘曲线复习提醒（已实现）

基于艾宾浩斯遗忘曲线，每道题按 **1 → 2 → 4 → 7 → 15 → 30 → 60 → 120 天** 的间隔自动排期复习，比「连续 2 次就标掌握」更科学——它贴合记忆衰减规律，记得越牢，间隔拉得越长。

配置每日定时任务（建议 20:00）后，到点只推送**今天到期**的题（含逾期），不再随机抽题：

```
📚 遗忘曲线复习时间到
今天有 3 道到期（间隔：1→2→4→7→15→30→60→120 天）

第 1 / 3 道 （第 1 次复习，距上次 1 天）
[判断推理 · 图形推理]
题目内容……
还记得解法吗？回复 记得 / 不记得
```

- 回复 **记得** → 推进到下一个更长间隔（如 1 天 → 2 天 → 4 天 …），并告诉你下次复习日期
- 回复 **不记得** → 间隔重置回第 1 天（明天再出现），多过一遍就熟了
- 走完整个周期（8 次）→ 自动标记该题「已掌握」，不再安排

> 存量旧题首次运行时会**错峰导入**（首个到期日分散到未来 30 天内），避免几百道题一次性涌入。每天最多推送 8 道，超出部分留在队列次日继续。

## 数据在哪

所有数据存在本地，默认：

```
~/.apexnotes/data/
├── daily/                ← 每天的套题成绩（daily/2026-07-17.json）
├── wrong_questions.json  ← 所有错题（累积）
├── knowledge_tree.json   ← 知识框架层级树
├── knowledge_flat.json   ← 知识框架扁平索引（含考点路径）
├── review_state.json     ← 遗忘曲线复习状态（每题排期/进度）
├── stats_cache.json      ← 7 日模块准确率等统计
├── backups/              ← 错题自动备份（每次写入前）
└── exports/              ← 导出的 Excel / HTML
```

- **备份**：整个 `~/.apexnotes/data/` 目录复制即可。每次写入错题前会自动备份到 `backups/`
- **改位置**：用环境变量 `APEXNOTES_DATA_DIR` 覆盖数据目录
- **多设备同步**：用云盘（iCloud / Dropbox）同步 `~/.apexnotes/data/` 目录

## ⚠️ 跨平台注意（Mac ↔ Windows 必读）

知识框架的 4 个 `.md` 和 `images/` 大量使用**中文文件名**。Mac 与 Windows 间用 zip 传递时，编码差异会导致文件名乱码、图片大量缺失。

**典型现象**：Mac 打包 → Windows 解压后，`言语理解.md` 变乱码，`images/错题/` 下图片大量丢失（如 580 张只解出 13 张）。

**根因**：Mac 的 `zip` 用 UTF-8 存文件名但不打 UTF-8 标志位，Windows 按本地编码（cp437）解读，中文全误读。

**正确做法（任选其一）**：
- 打包端（Mac）：用 Keka 勾选「UTF-8 编码」，或终端 `zip -r -X ApexNotes.zip ApexNotes/`
- 解压端（Windows）：用 **7-Zip** 解压，不要用系统自带解压
- 已损坏补救：用 Python 修复文件名 `name.encode('cp437').decode('utf-8')` 后重命名

**最佳实践**：用 **Git** 同步项目（`clone` / `pull`）彻底规避 zip 编码问题；数据目录 `~/.apexnotes/data/` 不进 Git，单独用云盘同步。

> 详细多平台接入与排障见 [`PLATFORM_GUIDE.md`](PLATFORM_GUIDE.md)，Agent 完整技术流程见 [`SKILL.md`](SKILL.md)。

## 常见问题

**Q：没有多模态模型，截图识别不了怎么办？**
A：把题目文字手动复制发给 agent，用文字方式录入，功能完全相同。

**Q：数据存在哪，别人能看到吗？**
A：默认存你电脑的 `~/.apexnotes/data/`，纯本地 JSON，不上传任何云端。分享项目时只分享文件夹，对方安装后建自己的空库。

**Q：发的错题归类错了能改吗？**
A：录入时确认卡片就可以改（回复「归类改成XX」）。已入库的题后续版本会支持编辑，当前可手动改 `wrong_questions.json`。

**Q：支持哪些考试？**
A：科目映射默认覆盖国考 / 省考行测，也适配事业单位职测、军队文职、选调生等（部分模块名需 Fork 微调）。
